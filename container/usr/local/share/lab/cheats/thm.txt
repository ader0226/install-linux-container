THM (TryHackMe) CHEAT SHEET
===========================

完整流程
  HOST 端：在含 xxx.ovpn 的目錄執行 install 腳本（一鍵）
    /bin/bash -c "$(curl -fsSL <INSTALL_URL>)"     # macOS / Linux
    iex "& { $(iwr -useb <INSTALL_URL>) }"         # Windows PowerShell

  腳本會自動：
    1. 檢查 docker compose 是否可用
    2. 找 host 一個閒置 port 當 SOCKS5 對外 port
    3. 啟動容器、容器內自動連 OpenVPN（讀 /vpn/*.ovpn）
    4. 容器內自動起 microsocks SOCKS5 proxy（容器內 :1080）
    5. 用 Chrome / Edge / Brave 開獨立 profile，帶 SOCKS5 proxy
    6. 把終端帶進容器互動 shell

  在容器內看靶機：browser 已透過 SOCKS5 連到 VPN 網段，
                  網址直接打 http://<TARGET_IP>/ 即可。

VPN 操作（容器內）
  vpn-up                    （重新）啟動 OpenVPN（讀 /vpn/*.ovpn 第一個）
  vpn-down                  斷開
  sudo tail -f /var/log/openvpn.log
  ip addr show tun0
  ip route | grep tun0

SOCKS5 proxy 操作（容器內）
  proxy-up                  （重新）啟動 microsocks
  proxy-down                停止 microsocks
  sudo tail -f /var/log/microsocks.log

把 .ovpn 放進來
  腳本會把當前目錄整個 mount 成 /vpn:ro，第一個 *.ovpn 自動使用。
  AUTH_FILE：若 .ovpn 需要帳密，把帳號/密碼存成 ./auth.txt 即可。

工具安裝（lab 預設沒裝，現場自己裝）
  sudo apt update
  sudo apt install -y nmap gobuster

預先打包好的 wordlist
  /usr/share/wordlists/dirbuster/directory-list-lowercase-2.3-medium.txt

掃描靶機（容器內，走 tun0 直連）
  nmap -sV -sC -p- -T4 <TARGET_IP>         全 port + 服務指紋 + 預設 script
  nmap -sV --top-ports 100 <TARGET_IP>     快速版（先看常見 100 個）
  gobuster dir -u http://<TARGET_IP> \
    -w /usr/share/wordlists/dirbuster/directory-list-lowercase-2.3-medium.txt
  curl -I http://<TARGET_IP>/              抓 HTTP banner

常見問題
  browser 連不到 SOCKS5
    → 容器是否還活著：docker ps | grep linux-container
    → 重新跑 install 腳本即可恢復。
  ping 不到靶機
    → THM 房間「Start Machine」要按下去；ip link show tun0 看 VPN 是否真上。
  AUTH_FAILED
    → THM 的 .ovpn 不需要帳密，多半是 .ovpn 下載到一半壞掉，重下一份。
